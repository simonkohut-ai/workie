## ðŸ› ï¸ MVP Setup & Local Run

> **NOTE:** This project requires a **Gemini API Key** and a **Firebase Service Account** for the backend to function.

### Step 1: Clone the Repository

```bash
git clone https://github.com/simonkohut-ai/workie.git
cd workie
```

### Step 2: Backend (AOS) Setup

1.  **Navigate** to the backend service directory:
    ```bash
    cd aos-service
    ```

2.  **Install Dependencies**:
    ```bash
    npm install
    ```

3.  **Create `.env`**: Create a file named `.env` inside `/aos-service` and add the required environment variables:
    ```env
    # --- REQUIRED FOR GEMINI API CALLS ---
    GOOGLE_API_KEY="YOUR_GEMINI_KEY_HERE"

    # --- REQUIRED FOR FIREBASE ADMIN SDK ---
    # Retrieve the JSON content of your Firebase Service Account Key File (as a single line string).
    # This grants the server access to Firestore.
    # In Vercel, this is handled via Environment Variables. 
    # Locally, you can use the path in config/firebase-admin.js or mock it if just testing agent logic.
    
    PORT=3001
    ```

### Step 3: Start the AOS Server

Run this command from the `/aos-service` directory:
```bash
npm start
```

*   **Success:** You will see `AOS Service running on port 3001`.
*   **Failure:** If the server crashes, check your `.env` file and ensure `GOOGLE_API_KEY` is set.

### Step 4: Frontend Setup

1.  **Navigate** to the client directory (in a new terminal):
    ```bash
    cd ../client
    ```

2.  **Install Dependencies**:
    ```bash
    npm install
    ```

3.  **Start React Dev Server**:
    ```bash
    npm run dev
    ```

### Step 5: Test the End-to-End Flow

1.  Ensure the AOS server is running (Step 3).
2.  Open the React app (usually `http://localhost:5173`).
3.  Click the **"Save & Test Agent"** button in the dashboard.
4.  **Monitor the AOS Console:** If successful, the server console will display the structured JSON output (Summary, Action Items, Email Draft) generated by the Gemini 3 Agent.

## ðŸš€ Deployment to Vercel

This project is configured for **Zero-Config Deployment** on Vercel.

1.  **Import Project:** Import the GitHub repository into Vercel.
2.  **Environment Variables:** Add the following in Vercel Project Settings:
    *   `GEMINI_API_KEY`: Your Google Gemini API Key.
    *   `FIREBASE_ADMIN_CREDENTIALS`: The JSON string of your Firebase Service Account.
    *   `APP_ID`: `workie-app-id`
3.  **Deploy:** Vercel will automatically build the React frontend and deploy the AOS API as serverless functions.
